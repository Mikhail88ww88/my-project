# Исследование рынка: AI-виджет онлайн-чата на сайт

> Контекст: AI-консультант для компании по поздравлениям ростовыми куклами.
> Бот работает 24/7, собирает номер телефона, передаёт лиды в AmoCRM.
> Дата исследования: март 2026 года.

## 1. КОНКУРЕНТЫ — 5 платформ для AI-виджета чата на сайт

### 1.1 Jivo (JivoChat)

**Что умеет:**
Полноценный омниканальный чат-виджет для сайта с поддержкой операторов и AI-автоматизацией. Поддерживает чат на сайте, мессенджеры, email, звонки — всё в одном интерфейсе. AI-оператор работает на базе LLM и может полностью заменить живого сотрудника в диалоге. Есть конструктор сценариев для чат-ботов (интеграция с SaluteBot). Поддерживает офлайн-форму сбора контактов.

**Плюсы:**
- Один из лидеров русскоязычного рынка, хорошая локализация
- Бесплатный тариф с базовыми функциями
- Готовая интеграция с AmoCRM через официальный коннектор
- Мобильное приложение для операторов
- AI-оператор уже обучен на вашем контенте

**Минусы:**
- AI-оператор стоит отдельно ~13 000 ₽/мес — дорого для малого бизнеса
- Сложно тонко кастомизировать логику бота без технических знаний
- Конструктор сценариев уступает специализированным no-code платформам

**Цена:**
- Бесплатно: базовый чат, до 5 операторов
- Базовый: от 742 ₽/оператор/мес
- Профессиональный: от 1 117 ₽/оператор/мес (включает чат-боты)
- Корпоративный: от 2 617 ₽/оператор/мес
- AI-оператор: ~13 000 ₽/мес отдельно

### 1.2 Carrot Quest

**Что умеет:**
Маркетинговая платформа с чатом на сайте, поп-апами, email-рассылками и чат-ботами. Основной акцент — автоматизация маркетинга и конверсия. Бот строится в визуальном конструкторе без кода: сценарии ветвления, сбор форм, квалификация лидов, передача диалога оператору. Трекинг поведения посетителей на сайте.

**Плюсы:**
- Мощный no-code конструктор сценариев с кнопками и формами
- Удобный визуальный редактор сценариев
- Сегментация пользователей по поведению на сайте
- Интеграция с AmoCRM

**Минусы:**
- Цена зависит от трафика сайта — при росте дорожает
- Нет встроенного LLM / GPT-диалога "из коробки" (нужна интеграция)
- Сложнее в настройке для нетехнического пользователя
- Дороговато для старта

**Цена:**
- От 1 000 ₽/мес за 1 000 посетителей (тариф "Коммуникации")
- Пробный период: 7 дней бесплатно
- Средний чек для активного сайта: 3 000–10 000 ₽/мес

### 1.3 Tidio

**Что умеет:**
Западная платформа с отличным виджетом живого чата и AI-агентом Lyro на базе GPT/Claude. Lyro автоматически отвечает на вопросы пользователей, опираясь на базу знаний сайта, и решает до 64–67% обращений без оператора. Есть конструктор chatflow (drag-and-drop), шаблоны сценариев, интеграции с e-commerce (Shopify, WooCommerce), 120+ интеграций.

**Плюсы:**
- Один из лучших AI-агентов на рынке (Lyro) — реально работает
- Бесплатный тариф с живым чатом
- Простая установка (1 скрипт на сайт)
- Шаблоны сценариев: сбор лидов, FAQ, обработка возражений

**Минусы:**
- Интерфейс и поддержка на английском (нет полноценного русского)
- Lyro AI — дополнительная плата за каждый разговор
- Интеграция с AmoCRM через Zapier/Make (нет прямой)
- Могут быть проблемы с оплатой российскими картами

**Цена:**
- Бесплатно: живой чат, 50 диалогов/мес
- Starter: ~$29/мес (100 диалогов)
- Growth: ~$59/мес (2 000 диалогов)

## 2. ФУНКЦИИ — что умеют AI-виджеты чата на сайт

### MVP — обязательные функции для запуска

1. **Виджет чата на сайте** — всплывающее окно чата (кнопка + диалог), настройка позиции, цвета, текста приветствия
2. **Автоприветствие** — бот первым пишет через N секунд после захода на сайт или на конкретную страницу
3. **Сценарный бот (chatflow)** — ветвление диалога: вопросы → варианты ответов кнопками → следующий шаг
4. **Сбор контактных данных** — форма запроса имени и номера телефона внутри диалога
5. **Передача лида в AmoCRM** — автоматическое создание сделки/контакта с номером телефона при заполнении формы
6. **Офлайн-форма** — если оператор недоступен, посетитель оставляет контакты
7. **Ответы на FAQ** — заранее прописанные ответы на типовые вопросы (цены, условия, география работы)
8. **Мобильная адаптация виджета** — корректное отображение на смартфонах
9. **Уведомления оператору** — оповещение (email, Telegram, звонок), когда пришёл новый диалог

### Расширенные функции (добавляются после MVP)

10. **LLM-диалог (GPT/Claude)** — свободный диалог на основе большой языковой модели, не требующий прописанных сценариев
11. **База знаний** — загрузка документов / FAQ / текста сайта, бот отвечает на вопросы по ней
12. **Квалификация лидов** — бот задаёт вопросы (дата мероприятия, город, количество гостей) и оценивает, насколько запрос горячий
13. **Смена оператора** — передача диалога от бота к живому менеджеру в один клик
14. **Аналитика и конверсии** — статистика: сколько диалогов, какой % оставил контакт, на какой вопрос чаще всего отвечают
15. **A/B-тест приветствий** — тестирование разных текстов автоприветствия для повышения конверсии
16. **Персонализация** — обращение по имени, запоминание истории предыдущих визитов
17. **Голосовой ввод / вывод** — интеграция speech-to-text для ответа голосом
18. **Мультиязычность** — автоматическое определение языка посетителя и ответ на нём
19. **Proactive triggers** — разные сценарии на разных страницах (страница цен, страница "контакты", страница конкретной услуги)
20. **Интеграция с календарём** — запись на встречу/созвон прямо из виджета

## 3. UX И СЦЕНАРИИ — как посетитель взаимодействует с виджетом

### Сценарий 1: "Зашёл ночью узнать цену"

**Ситуация:** Пользователь нашёл сайт в 23:30 после запроса в Яндексе "ростовая кукла на день рождения Ростов-на-Дону цена". Операторов нет.

**Идеальный сценарий с AI-ботом:**
1. Через 5 секунд после захода — всплывает виджет с сообщением: "Привет! Я Маша, AI-ассистент. Помогу узнать стоимость и наличие костюмов. С чего начнём?"
2. Пользователь пишет: "Сколько стоит ростовая кукла на 2 часа?"
3. Бот отвечает конкретно: диапазон цен, зависимость от типа костюма и времени
4. Пользователь: "А есть ли Человек-паук?"
5. Бот: "Да, есть! Давайте уточним детали — оставьте номер телефона, и наш менеджер свяжется с вами завтра утром и подтвердит наличие."
6. Пользователь оставляет номер — лид уходит в AmoCRM, тег "ночной", менеджер видит заявку утром.

**Что важно:** бот не должен говорить "Мы перезвоним вам в рабочее время" без сбора контакта — это потеря лида. Сначала контакт, потом обещание.

---

### Сценарий 2: "Хочет срочно заказать сегодня"

**Ситуация:** Пользователь зашёл в 11:00, день рождения ребёнка сегодня в 17:00. Ищет кого-то срочно.

**Идеальный сценарий:**
1. Бот приветствует: "Добрый день! Помогу найти свободного аниматора."
2. Пользователь: "Мне нужна кукла СЕГОДНЯ в 17:00, возможно?"
3. Бот: "Понял, срочный запрос! Уточните, пожалуйста — ваш город и какой персонаж нужен? Я сразу передам вашу заявку менеджеру."
4. Пользователь вводит данные — бот берёт имя + телефон + город + персонаж
5. Лид уходит с тегом "СРОЧНО" в AmoCRM, менеджеру приходит push-уведомление немедленно
6. Бот: "Отлично! Менеджер позвонит вам в течение 5 минут. Пока ожидаете — вот наше портфолио работ: [ссылка]"

**Что важно:** бот должен понимать слово "срочно" / "сегодня" и реагировать иначе — не стандартным сценарием, а ускоренным. Это делается через LLM-логику.

---

### Сценарий 3: "Сомневается и сравнивает"

**Ситуация:** Пользователь читает страницу "О нас" 3 минуты, открывал несколько вкладок с конкурентами. Ещё не написал.

**Идеальный сценарий:**
1. Триггер: после 2 минут на странице — всплывает ненавязчивое сообщение: "Выбираете аниматора для праздника? Могу помочь с выбором — расскажите, что важно для вашего мероприятия?"
2. Пользователь: "Я смотрю несколько компаний, у вас цена выше чем у конкурентов"
3. Бот: "Понимаю! Разница в цене часто объясняется качеством костюмов и опытом аниматоров. Расскажите — это для какого возраста детей и сколько человек? Подберу оптимальный вариант."
4. Бот мягко квалифицирует лид: дата, количество, бюджет
5. Предлагает: "Хотите, наш менеджер позвонит вам и расскажет, чем мы отличаемся? Обычно после звонка вопросы исчезают сами собой :)"
6. Пользователь оставляет номер.

---

### Что раздражает пользователей в виджетах

- **Агрессивный поп-ап через 1 секунду** — люди не успели ничего прочитать, а уже требуют данные
- **"Операторы недоступны" без альтернативы** — нет формы, нет бота, просто заглушка
- **Бот не понимает свободный текст** — только кнопки, на любое нестандартное сообщение: "Не понял, выберите из меню"
- **Просьба ввести email вместо телефона** — в России WhatsApp важнее email, лиды теряются
- **Виджет перекрывает контент** — особенно на мобильных
- **Долгий первый ответ** — пользователь написал, ждёт 10 секунд, закрывает вкладку

### Что работает хорошо

- Персональное приветствие с именем и контекстом страницы
- Быстрые кнопки-подсказки ("Узнать цену", "Проверить наличие", "Записаться")
- Прогресс-индикатор при заполнении формы ("Осталось 1 вопрос")
- Подтверждение после оставленного номера с конкретным временем звонка

## 4. ТЕХНОЛОГИИ — варианты реализации AI-виджета

### Вариант 1: Готовый виджет (Jivo / Chaport / Tidio)

**Как работает:** Устанавливаешь готовый JS-скрипт на сайт, настраиваешь в личном кабинете платформы, подключаешь AmoCRM через официальную интеграцию или Zapier.

**Плюсы:**
- Запуск за 1–3 дня
- Не нужен программист (настройка через UI)
- Хостинг, безопасность, масштабирование — на платформе
- Мобильные приложения для операторов включены
- AI-функции (если есть) уже встроены и проверены

**Минусы:**
- Ограниченная кастомизация диалоговой логики
- Нет глубокой LLM-интеграции (кроме Jivo AI за доп. плату)
- Зависимость от вендора: цена, условия, закрытие сервиса
- Бренд платформы в виджете (нужно платить за убирание)

**Стоимость запуска:** 0–5 000 ₽ (настройка)
**Ежемесячные расходы:** 1 500–15 000 ₽/мес (в зависимости от тарифа и AI)

---

### Вариант 2: Своя разработка с LLM (Claude / GPT-4o)

**Как работает:** Разработчик создаёт кастомный виджет (JS-компонент), бэкенд на Node.js/Python, интегрирует OpenAI API или Claude API, пишет промпт с базой знаний о компании, интегрирует с AmoCRM через API.

**Архитектура MVP:**
- Frontend: React/Vue виджет (чат-пузырь + диалог)
- Backend: API-сервер (FastAPI или Express)
- LLM: OpenAI gpt-4o-mini или Claude 3 Haiku
- База знаний: системный промпт + RAG на документах
- Интеграция: AmoCRM REST API

**Плюсы:**
- Полный контроль над UX, логикой, брендингом
- Не зависишь от платформы — своё, можно перепродавать
- LLM-диалог значительно лучше, чем сценарные боты
- Можно тонко настраивать: знает всё о компании, не галлюцинирует вне темы

**Минусы:**
- Нужен разработчик: 2–6 недель работы
- Поддержка, хостинг, обновления — на тебе
- Стоимость LLM-API растёт с количеством диалогов
- Риски безопасности требуют дополнительной работы

**Стоимость запуска:** 100 000–350 000 ₽ (разработка под ключ)
**Ежемесячные расходы:**
- Хостинг (VPS/облако): 1 000–3 000 ₽/мес
- LLM API (при 300 диалогов/мес): ~500–2 000 ₽/мес (gpt-4o-mini: ~$0.30/1M токенов)
- Итого: ~2 000–5 000 ₽/мес при небольшом трафике

---

### Вариант 3: Гибрид — готовая платформа + кастомный LLM-бэкенд

**Как работает:** Берётся готовый виджет (Jivo, Chaport, или даже open-source решение типа Chatwoot) для front-end части (красиво, работает на мобилах). Но вместо встроенного бота — подключается внешний webhook/API на свой LLM-сервер. Виджет передаёт сообщение на твой бэкенд, бэкенд отвечает через GPT/Claude, ответ приходит обратно в виджет.

**Плюсы:**
- Лучший UX виджета (готовый, протестированный)
- Мощный AI (LLM) за кастомным API
- Быстрый запуск: 1–3 недели
- Гибкость: логику бота можно менять без переписывания виджета

**Минусы:**
- Нужен бэкенд-разработчик для LLM-части
- Зависимость от платформы (виджет) + от API (LLM)
- Не все платформы поддерживают webhook для ответов от внешнего бота

**Стоимость запуска:** 50 000–150 000 ₽ (разработка бэкенда + настройка)
**Ежемесячные расходы:**
- Платформа (Jivo Pro): ~3 000–5 000 ₽/мес
- LLM API: ~500–3 000 ₽/мес
- Хостинг бэкенда: 1 000–2 000 ₽/мес
- Итого: ~5 000–10 000 ₽/мес

## 5. МОНЕТИЗАЦИЯ — как зарабатывать на продаже виджета как услуги

### Модель 1: Разовая разработка под ключ

Создаёшь AI-виджет для бизнеса под ключ — настраиваешь под их компанию, обучаешь на их FAQ, подключаешь AmoCRM.

**Ценовой диапазон:**
- Простой сценарный бот + виджет: 35 000–80 000 ₽
- AI-виджет с LLM + AmoCRM + обучение: 120 000–300 000 ₽
- Полная разработка с нуля (кастомный код): от 300 000 ₽

**Плюсы:** большие разовые выплаты, простая продажа
**Минусы:** нет повторяющегося дохода, нужно постоянно искать новых клиентов

---

### Модель 2: Ежемесячная подписка (SaaS)

Разрабатываешь один продукт (виджет-платформу) и продаёшь доступ к нему по подписке разным бизнесам.

**Рыночные ориентиры:**
- Базовый тариф (виджет + сценарный бот + AmoCRM): 2 000–5 000 ₽/мес
- Стандарт (+ LLM-диалог, обучение на FAQ): 5 000–12 000 ₽/мес
- Про (несколько сайтов, приоритет, аналитика): 15 000–30 000 ₽/мес

**Плюсы:** предсказуемый доход, масштабируется, актив растёт
**Минусы:** нужна инфраструктура, поддержка, высокие требования к надёжности

---

### Модель 3: Плата за лиды / за диалог

Берёшь процент от закрытых лидов или фиксированную плату за каждый диалог с номером телефона.

**Варианты:**
- 200–2 000 ₽ за квалифицированный лид (с телефоном и намерением купить)
- 15–50 ₽ за диалог (любой, в котором пользователь написал более 1 сообщения)

**Плюсы:** легко продать ("платите только за результат"), низкий порог входа для клиента
**Минусы:** сложно контролировать качество, клиент может оспаривать лиды, нужна аналитика

---

### Модель 4: Гибридная (разовое внедрение + подписка)

Самая жизнеспособная модель на практике (используют большинство агентств и indie-разработчиков).

**Структура:**
1. **Разовое внедрение:** 50 000–150 000 ₽ (настройка, обучение бота, подключение AmoCRM, тестирование)
2. **Ежемесячная поддержка:** 5 000–15 000 ₽/мес (обновление базы знаний, API-доступ к LLM, хостинг, техподдержка)
3. **Опциональные допродажи:** расширение на новые сценарии, новые интеграции, вторые сайты

**Почему работает:**
- Клиент понимает: сначала платит за настройку, потом за стабильную работу
- Разработчик получает гарантированный ежемесячный доход
- При 10 клиентах на подписке по 8 000 ₽ = 80 000 ₽/мес пассивно

---

### Модель 5: White-label перепродажа

Покупаешь доступ к платформе (например, Tidio или кастомное решение) оптом и перепродаёшь под своим брендом малому бизнесу с наценкой.

**Пример:** берёшь Tidio за $59/мес, продаёшь клиенту "ИИ-ассистент для сайта" за 8 000 ₽/мес — маржа ~50%.

**Плюсы:** нет разработки, быстрый старт, масштабируется
**Минусы:** зависимость от вендора, ограниченная кастомизация, вендор может поднять цены

---

### Партнёрства

- **AmoCRM партнёрская программа:** официальный партнёр AmoCRM получает % с привлечённых клиентов и скидки на внедрение. Актуально для агентств, которые делают боты + CRM под ключ.
- **Аниматорские агентства / франшизы:** предложить готовый AI-виджет всем студиям, которые работают по франшизе — тиражируемое решение.
- **Маркетинговые агентства:** реселлинг виджета в пакете с лидогенерацией.

## Итоговые выводы

1. **Для вашего конкретного MVP** (кукольная компания, 1 сайт, AmoCRM) — оптимальный выбор: **Jivo на тарифе Профессиональный (~3 000 ₽/мес)** с настройкой сценарного бота. Запуск за 3–5 дней без программиста. Если нужен GPT-диалог — **гибрид: Chaport/Jivo + кастомный LLM-бэкенд** (~50–100 тыс. ₽ разработки).

2. **Ключевые функции для MVP:** приветствие → ответы на FAQ (цены, персонажи, города) → сбор имени + телефона → передача в AmoCRM → уведомление менеджеру.

3. **Главная UX-ошибка** в нише услуг: бот отвечает "напишите нам" или "перезвоним в рабочее время" без сбора контакта. Ночью уходит 30–40% потенциальных лидов именно так.

4. **Если цель — продавать виджет другим бизнесам:** гибридная модель (разовое внедрение 80–150 тыс. ₽ + подписка 5–15 тыс./мес) — самая рабочая. Выход на безубыток при 8–10 клиентах.

5. **Технологически** оптимален GPT-4o-mini или Claude 3 Haiku для диалогов — дёшево (~$0.15–0.30 за 1M токенов), достаточно умно для консультаций.
